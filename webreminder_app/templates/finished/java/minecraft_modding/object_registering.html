Теория

Для регистрации почти любого объекта (блока, предмета, инструмента и проч.) мы должны:

  1. Создать регистратор объектов: public static final DeferredRegister<T> T = DeferredRegister.create(ForgeRegistries.T, mod_id);
  2. Зарегистрировать в регистраторе объект: public static final RegistryObject<T> T_ = registerBlock("id", () -> TFabric; );
  3. Добавить регистратор в мод: T.register(eventBus)

Почему такая ебля?

  • Создаем объект регистратора, (его родителем должен являться соответсвующий реестр) в который будем складывать объекты. Это отложенный регистратор, который зарегистрирует наши предметы в нужный момент, в ином случае все может наебнуться. Про нужные моменты смотри статью про события в майнкрафте
  • Затем регистрируем в него предмет. Мы не вызываем просто метод, а сохраняем его значение, так как нам потом нужен доступ к объекту, который мы зарегистрировали. Это происходит из-за того, что мы регистрируем не сам объект, а его фабрику, которая вызовется в нужный момент отложенным регистратором
  • Затем добавляем регистратор в наш мод, передавая ему объект класса событий, что позволяет ему добавлять в нужный момент

Какие объекты можем регистрировать

Здесь неполный список

  • Block: блоки
  • Item: предметы
  • BlockEntityType: сущности блоков
  • EntityType<?>: мобы и сущности
  • MenuType<?>: GUI-меню (контейнеры)
  • RecipeSerializer<?>: сериализаторы рецептов
  • RecipeType<?>: типы рецептов
  • SoundEvent: звуковые события
  • Potion: зелья
  • Effect: эффекты (например, скорость, сила)
  • Enchantment: чары
  • Attribute: атрибуты (например, здоровье, урон)
  • CreativeModeTab: вкладки творческого режима
